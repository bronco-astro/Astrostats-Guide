Chapter 11 — Time Series Analysis

Time series analysis is about data collected over time. In astronomy, this usually means observing the same object many times and recording how a quantity changes.

Examples include light curves, radial velocity measurements, or count rates. In time series data, the order of observations matters.

Why time series are different

In time series data, measurements close in time are often related. This means data points are not independent. Many methods from earlier chapters assume independence, so they do not apply directly here.

Time series analysis focuses on understanding and modeling this time dependence.

First step: look at the data

The first thing to do is always to plot the data against time. Plotting helps identify trends, sudden changes, gaps, outliers, and possible periodic behavior.

Trends and variability

A time series often contains both trends and variability. A trend is a slow change over time, such as a gradual increase or decrease. Variability refers to fluctuations around a typical level.

Removing trends is often necessary before modeling short-term behavior.

Stationarity

Many time series models assume stationarity. A stationary time series has statistical properties that do not change with time. The mean and variance stay roughly constant.

Real astronomical time series are often not stationary. Transformations such as differencing are used to make data closer to stationary.

Autocorrelation

Autocorrelation measures how similar the data are to themselves at different time lags. If values close in time are similar, autocorrelation at small lags is large.

Autocorrelation describes memory in the data and is a key idea behind many time series models.

ARIMA models

ARIMA stands for AutoRegressive Integrated Moving Average. ARIMA models describe time series using past values, past errors, and differencing.

An ARIMA model is written as ARIMA(p, d, q).

Autoregressive (AR) part

The autoregressive part models the current value using previous values. In an AR model, the value at time t depends on values at earlier times.

The parameter p specifies how many past values are used. Larger p means longer memory.

Integrated (I) part

The integrated part refers to differencing the data. Differencing removes trends and helps make the time series stationary.

The parameter d specifies how many times differencing is applied.

Moving average (MA) part

The moving average part models the current value using past errors. Instead of using past values directly, the MA part uses previous prediction errors.

The parameter q specifies how many past errors are included.

Choosing p, d, and q

Choosing ARIMA parameters is an important step. Plots of the autocorrelation function and partial autocorrelation function are often used to guide this choice.

The goal is to find a model that captures structure without overfitting noise.

Fitting an ARIMA model

Fitting an ARIMA model means estimating its parameters from data. This is usually done using maximum likelihood.

The fitted model represents a balance between matching the data and keeping the model simple.

Model diagnostics

After fitting an ARIMA model, residuals should be examined. If the model is appropriate, residuals should look like uncorrelated noise.

Patterns in residuals suggest the model is missing structure. Diagnostics are essential before trusting forecasts.

Forecasting

ARIMA models are often used for forecasting. Forecasts are more reliable over short time horizons. Uncertainty increases quickly when predicting far into the future.

Limits of ARIMA models

ARIMA models work well for many types of data, but they have limits. They assume linear relationships and may not capture complex periodic signals or nonstationary behavior.

In astronomy, ARIMA is often one tool among many.
